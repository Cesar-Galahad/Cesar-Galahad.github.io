La matriz de una aseguradora recibe diariamente cerca de 100 solicitudes de diversos tipos de seguros (auto, vida, desempleo, etc.), pero el proceso actual no está automatizado, lo que provoca retrasos en el seguimiento y atención al cliente. Actualmente, cada encargado registra la información en hojas de cálculo individuales, lo que dificulta la gestión eficiente. Cada solicitud incluye datos del seguro, del cliente asegurado y de la sucursal emisora. Las sucursales cuentan con agentes encargados de captar clientes, quienes reciben comisiones variables según el tipo de seguro. Sin embargo, no existe un control adecuado sobre qué agente genera cada solicitud, lo que provoca conflictos como duplicación de registros y ofrecimiento de seguros a clientes ya atendidos. Las solicitudes se agrupan por tipo de seguro y se envían diariamente a la matriz para su registro, pero los agentes no tienen visibilidad del estado de estas, lo que agrava la duplicación de esfuerzos. La aseguradora busca contratar una empresa desarrolladora de software para implementar un sistema automatizado, con especial énfasis en la seguridad de acceso (cuentas individuales por empleado), el seguimiento de movimientos y el cumplimiento de políticas e imagen empresarial.

Nuestra propuesta para la problemática de NexoSeguros (Tu nexo con la seguridad), es realizar un sistema para la automatización de entrada, donde se pueda visualizar mediante una base de datos, las solicitudes, clientes registrados y realice una comparación para verificar y evitar que los agentes de seguros no registren un mismo cliente. También se realiza el programa para una mejor gestión de estos datos, donde se muestran los servicios que ofrecen como aseguradora y por último, la creación de una red para la conexión de nuestra matriz con la sucursal
